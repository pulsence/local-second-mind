# Commit Message Format

This document defines the required commit message format for Local Second Mind.

## Commit Message Template

```
Added [feature name]

Feature Name:
- Bullet point describing capability
- Another capability
- Configuration options added
```

## Guidelines

- Use present tense ("Added" not "Adding")
- First line is a brief summary of the change
- Follow with a blank line, then bullet points describing:
  - What capability was added
  - Any other notable changes
  - Configuration options that were added/modified

## Example

```
Added research agent multi-source query

Research Agent:
- Added ability to query multiple remote sources in parallel
- Added configurable source priority ordering
- Added source-specific timeout configuration
```
